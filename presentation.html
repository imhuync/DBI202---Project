Giới thiệu dự án

  • Dự án tập trung vào việc số hóa quy trình vận hành của một khách sạn quy mô vừa và lớn.
  • Giải quyết các vấn đề về quản lý phòng trống, đặt phòng chồng chéo và tính toán hóa đơn phức tạp.
  • Đặc điểm nổi bật: Toàn bộ thao tác với dữ liệu (Business Logic) được viết bằng MSSQL.
  • Công nghệ cốt lõi: SQL Server cho tầng dữ liệu và Modern Web cho tầng ứng dụng.

Tính năng chính

Lưu trú

Quản lý đặt phòng (Booking), nhận phòng (Check-in) và trả phòng (Check-out) nhanh chóng.

Dịch vụ

Tích hợp gọi món, giặt là, spa vào trực tiếp hóa đơn phòng của khách hàng.

Tài chính

Tự động hóa tính toán hóa đơn, thuế VAT và các khoản chiết khấu/đặt cọc.

Công nghệ sử dụng

  • Backend (MSSQL): 100% logic xử lý tại Stored Procedures giúp tối ưu hiệu năng và bảo mật.
  • Server (Node.js): Đóng vai trò cầu nối (Bridge), chỉ gọi SP và không chứa logic tính toán.
  • Frontend: Vite + TypeScript + Vanilla CSS tạo trải nghiệm mượt mà, tối ưu hiệu năng.
  • Data Security: Ràng buộc dữ liệu (Constraints) và Transaction được xử lý triệt để tại DB.

Thiết kế Cơ sở dữ liệu

Hệ thống bao gồm 9 bảng chính được thiết kế chuẩn hóa để tránh dư thừa dữ liệu:

Core

ROOM, ROOM_TYPE, GUEST

Operational

BOOKING, BOOKING_DETAIL

Financial

SERVICE, SERVICE_USAGE, INVOICE, EMPLOYEE

Mối quan hệ (Entity Relationship)

  • GUEST - BOOKING: Mối quan hệ 1-N (Một khách hàng có thể có nhiều lần đặt phòng).
  • BOOKING - ROOM: Thông qua bảng trung gian BOOKING_DETAIL để hỗ trợ đặt nhiều phòng trong một đơn.
  • SERVICE_USAGE: Liên kết với BOOKING_DETAIL để biết dịch vụ được dùng bởi phòng nào, khách nào.
  • INVOICE: Liên kết 1-1 với BOOKING để tổng kết toàn bộ chi phí.

Quản lý phòng (ROOM)

CREATE TABLE ROOM (
    room_id INT IDENTITY PRIMARY KEY,
    room_number NVARCHAR(10) UNIQUE,
    status NVARCHAR(20) CHECK (...), -- Clean, Dirty, Occupied
    floor INT
);
  • Mỗi phòng thuộc một loại phòng (ROOM_TYPE) quy định giá cơ bản và sức chứa.
  • Trạng thái phòng tự động cập nhật khi khách Check-in hoặc Check-out.

Khách hàng & Nhân viên

GUEST

Lưu trữ: Họ tên, Email, CCCD (Unique), Quốc tịch. Mã định danh hỗ trợ khách hàng thân thiết.

EMPLOYEE

Quản lý: Tài khoản đăng nhập, Vai trò (Receptionist, Manager, Housekeeper).

Bảo mật

Sử dụng Password Hashing và Role-based Access Control (RBAC).

Quy trình Đặt phòng

  • BOOKING: Lưu thông tin chung như ngày tạo, nhân viên phụ trách, tiền cọc.
  • BOOKING_DETAIL: Lưu chi tiết từng phòng, giá tại thời điểm đặt (tránh biến động giá sau này).
  • Hỗ trợ trạng thái: Pending -> Confirmed -> Completed/Cancelled.

Dịch vụ & Tiện ích

CREATE TABLE SERVICE_USAGE (
    usage_id INT PRIMARY KEY,
    booking_detail_id INT, -- Link to room
    service_id INT,
    quantity INT,
    total_price DECIMAL(12,2)
);

Mỗi khi khách hàng sử dụng dịch vụ, hệ thống sẽ log lại thời gian và tính toán thành tiền tức thì dựa trên đơn giá tại bảng SERVICE.

Hệ thống Hóa đơn (INVOICE)

  • Tự động tổng hợp room_charge (tiền phòng) và service_charge (tiền dịch vụ).
  • Áp dụng thuế suất mặc định (10% VAT).
  • Hỗ trợ nhiều phương thức thanh toán: Tiền mặt, Thẻ, Chuyển khoản, Ví điện tử.

Nghiệp vụ: Tính toán hóa đơn

Thủ tục sp_GenerateInvoice là "bộ não" của hệ thống tài chính:

-- Logic tính toán trong SP:
SET tax_amount = (room_charge + service_charge) * 0.1;
SET final_amount = (room_charge + service_charge + tax) - discount;

Nó quét qua toàn bộ BOOKING_DETAIL và SERVICE_USAGE liên quan để đưa ra con số cuối cùng chính xác nhất.

Kiến trúc Ứng dụng

  • DB-Centric Architecture: SQL Server giữ vai trò là "nguồn sự thật duy nhất".
  • Node.js Bridge: Backend gọn nhẹ, chỉ chịu trách nhiệm định tuyến (Routing) và xác thực.
  • Mạnh mẽ: Dễ dàng thay đổi Framework Frontend/Backend mà không làm hỏng Logic nghiệp vụ tại DB.
